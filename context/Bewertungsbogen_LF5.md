> Bewertungsgrundlage für die Leistungsfeststellung im Lernfeld 5.
> Die Gesamtbewertung setzt sich aus den beiden Teilen **Präsentation** und **Fachgespräch** zusammen.

## Teil 1: Präsentation

| Kriterium | Feedback (Erwartung) | Kommentar | erreichte Punkte | max. Punkte |
|---|---|---|:---:|:---:|
| **Inhalt** | sachlich richtig · fachlich vollständig · kundenorientiert (Zielgruppe) | | | 5 |
| **Nutzung** | Wie kann der Kunde die implementierte Datenbank nutzen? · Wie sieht die Nutzer-Schnittstelle des Kunden aus? · Was sind nächste wichtige Schritte für die Weiterentwicklung? | | | 5 |

## Teil 2: Fachgespräch

> **Wichtig:** Nur 3 der 5 Bereiche werden geprüft.
> Die beiden markierten Bereiche (⭐) sind Pflicht, den 3. Bereich dürft ihr wählen.

| Kriterium | Feedback (Erwartung) | Kommentar | erreichte Punkte | max. Punkte |
|---|---|---|:---:|:---:|
| ⭐ **Grundverständnis Datenbanken** | Was ist eine Datenbank? · Datenbankarten · Vor- und Nachteile relationaler DB | | | 10 |
| **Normalisierung** | 1NF, 2NF, 3NF – Kriterien, Anomalien, Zerlegung | | | 10 |
| ⭐ **Datenmodellierung (ER-Modell)** | Verständnis Schlüsselkonzepte (Primär- und Fremdschlüssel) und Kardinalitäten (1:1, 1:n, n:m) | | | 10 |
| **SQL-Verständnis** | SELECT, WHERE, GROUP BY, ORDER BY, INNER/LEFT JOIN | | | 10 |
| **Sicherheit und Rechte** | Bewusstsein für Datenschutz und Sicherheit bei DB · Zugriffskontrollen · Backups und Wiederherstellung · SQL-Injection | | | 10 |
| **Gesamt** | | | | **30** |

> Hinweis: Da nur 3 der 5 Bereiche geprüft werden, sind maximal **30 Punkte** (3 × 10) erreichbar.
